M04 – Local Storage

01

Introdução

  • A Web Storage consiste na capacidade de recorrer ao Web browser para armazenar e obter dados
  • O objeto localStorage fornece acesso a um armazenamento local
  • Permite adicionar, ler, modificar e remover dados desse domínio local (web browser)
  • Os dados são armazenados de forma persistente e não são excluídos quando fecha o navegador
02

Porquê usar Web Storage?

  • Rapidez e facilidade de acesso aos dados
  • Não requer tráfego de rede
  • Não sobrecarrega o servidor
  • Maior rapidez no acesso aos dados
  • Funcional mesmo em offline (porque dados estão no browser)

Soluções:

  • Cookies
  • Web Storage API
  • IndexedDB API
  • Bibliotecas (Lockr, localForage, Dexie, etc.)
03

Web Storage API

  • API para armazenamento persistente, no browser do cliente, de dados sob a forma de key-value
  • Storage size: Pode armazenar até 5MB de dados na local Storage
  • Dados nunca são transferidos para o servidor
  • local Storage vs session Storage: são 2 áreas diferentes, distinguem-se pelo âmbito e persistência de dados
04

Local Storage vs Session Storage

Característica Local Storage Session Storage
Acesso Todas as páginas podem guardar e aceder aos mesmos dados Limitado à janela/tab atual
Persistência Os dados persistem mesmo após fechar o browser Dados são removidos quando fecha o tab ou janela
Objeto window.localStorage ou localStorage window.sessionStorage ou sessionStorage
Associação Domínio/protocolo/porta Objeto janela/tabulador
05

Propriedades e Métodos

// Propriedades localStorage.length // Retorna o número de itens armazenados // Métodos localStorage.setItem(key, value) // Armazena um item localStorage.getItem(key) // Obtém um item localStorage.removeItem(key) // Remove um item localStorage.clear() // Remove todos os itens localStorage.key(index) // Obtém o nome da chave no índice especificado
06

Verificar suporte do navegador

if (typeof(Storage) !== "undefined") { // Código para localStorage/sessionStorage } else { alert("Seu navegador não suporta Web Storage"); }
07

Armazenar dados

localStorage.setItem("nome", "João Silva"); localStorage.setItem("idade", "30");
08

Ler dados

const nome = localStorage.getItem("nome"); console.log(nome); // "João Silva"
09

Armazenar objetos e arrays

Local Storage não armazena arrays, apenas dados primitivos sob a forma de strings
const usuario = { nome: "Maria", email: "maria@exemplo.com", idade: 25 }; // Converter para string antes de armazenar localStorage.setItem("usuario", JSON.stringify(usuario)); // Converter de volta para objeto ao ler const usuarioArmazenado = JSON.parse(localStorage.getItem("usuario"));
10

Métodos JSON

  • JSON.stringify() - converte um objeto JavaScript e transforma-o num JSON string
  • JSON.parse() - converte uma JSON string e transforma-o num objeto JavaScript
11

Verificar se um item existe

if (localStorage.getItem("usuario") !== null) { console.log("Item existe na localStorage"); } else { console.log("Item não existe"); }
12

Iterar pela Local Storage

for (let i = 0; i < localStorage.length; i++) { const key = localStorage.key(i); const value = localStorage.getItem(key); console.log(`${key}: ${value}`); }
13

Remover dados

// Remove um item específico localStorage.removeItem("nome"); // Remove todos os itens localStorage.clear();
14

Vantagens

  • Suportado em quase todos os navegadores, incluindo iOS e Android
  • Muito simples, fácil de usar!
15

Desvantagens

  • Síncrono
  • Suporta apenas formato de string
  • Serialização-desserialização é um processo custoso. Torna o processamento mais lento
  • A pesquisa nunca é ótima; pode ter uma queda visível de desempenho em caso de dados grandes
